# Kratos — Training Guide

## Theory

Kratos uses a QLoRA fine-tuned **Qwen2.5-Coder-7B-Instruct** model. Training teaches the base model to produce structured `<tool_call>` JSON tags and reason through pentesting methodology (recon → exploit → privesc). The dataset comes from real CTF writeups converted to multi-turn ChatML conversations, plus synthetic scenarios generated by a larger model.

Training runs on **Google Colab Pro** (A100 40GB) in ~20 min. The result is a Q5_K_M GGUF file (~5.4 GB) imported into Ollama.

## Steps

### 1. Prepare raw data

Collect CTF writeups (HTML, Markdown, or PDF) into `kratos-model/data/raw/`. We used 83 HackTheBox writeups from [0xdf.gitlab.io](https://0xdf.gitlab.io).

```bash
# Place writeup files in:
kratos-model/data/raw/*.html
```

### 2. Generate training dataset

```bash
# Convert writeups to ChatML conversations
uv run python kratos-model/scripts/prepare_dataset.py

# Generate synthetic CTF scenarios (optional, improves quality)
uv run python kratos-model/scripts/generate_synthetic.py

# Output: kratos-model/data/processed/all_train.jsonl
```

Each conversation follows the schema in `kratos-model/data/schema.py`:
```
system → user → assistant (with <tool_call>) → tool (<tool_result>) → assistant → ...
```

### 3. Upload notebook to Google Colab

Open `kratos-model/notebooks/train_colab.ipynb` in [Google Colab](https://colab.research.google.com/).

Select **A100 GPU** runtime (Colab Pro required).

### 4. Run training on Colab

Execute cells in order:

| Step | What it does | Time |
|------|-------------|------|
| 0 | Mount Google Drive + verify GPU | 10s |
| 1 | Install Unsloth | 2 min |
| 2 | Upload `all_train.jsonl` | manual |
| 3 | Load and inspect data | 5s |
| 4 | Load Qwen2.5-Coder-7B-Instruct (4-bit) | 2 min |
| 5 | Add LoRA adapters (r=64, alpha=128) | 5s |
| 6 | Format dataset to ChatML | 5s |
| 7 | **Train** (3 epochs, QLoRA, bf16) | ~15 min |
| 8 | Save LoRA to Google Drive | 1 min |
| 9 | Quick inference test | 30s |
| 10 | Export GGUF Q5_K_M (local, then copy to Drive) | 10 min |
| 11 | Create Ollama Modelfile | 5s |

**Key training parameters:**
- QLoRA: r=64, alpha=128, dropout=0.05
- Batch: 2 × 4 gradient accumulation = effective 8
- LR: 2e-4, cosine scheduler, warmup 5%
- Optimizer: adamw_8bit
- **bf16=True** (not fp16 — A100 supports bfloat16)
- Packing: enabled (fits more data per batch)

### 5. Download from Google Drive

After training, download from `Drive > kratos-model > gguf/`:
- `qwen2.5-coder-7b-instruct.Q5_K_M.gguf` (~5.4 GB)
- `Modelfile`

### 6. Import into Ollama

```bash
cd ~/Downloads  # or wherever you downloaded the files
ollama create kratos -f Modelfile
ollama run kratos  # test it
```

### 7. Connect to Kratos

```bash
# Set in .env
KRATOS_MODEL=kratos

# Run
uv run kratos --target 10.10.10.50
```

## Tips

- **More data = better model.** 89 conversations is a minimal dataset. Target 500+ for production quality.
- **Colab disconnects?** LoRA saves to Drive immediately after training (Step 8). Re-mount Drive and resume from GGUF export.
- **Want to iterate?** Edit `prepare_dataset.py` to extract more tool calls, re-run, re-train.
- **Different base model?** Change `BASE_MODEL` in the notebook. DeepSeek-Coder-V2-Lite or Qwen2.5-14B also work.
